# langlearning

A hands-on project for learning **LangChain**, **RAG pipelines**, **tool
calling**, and **LangGraph** by building a real agent, one stage at a time.

The end result: a CLI assistant that answers questions either by
retrieving from a local knowledge base (RAG) or by calling a tool
(calculator, current date/time, web search), routed dynamically by a
LangGraph agent.

The knowledge base itself is study notes on these exact concepts — see
`data/docs/` — so querying the assistant doubles as revision material.

## Why you don't need an API key for most of this

Anthropic doesn't offer an embeddings API, so this project's RAG pipeline
embeds text locally with a free `sentence-transformers` model — no key, no
network call, no cost. That means **Stages 1–3 below run with zero API
key**. Only Stage 4 (the LangGraph agent) calls Claude and needs
`ANTHROPIC_API_KEY`.

## Setup

```bash
python -m venv .venv
# Windows:
.venv\Scripts\activate
# macOS/Linux:
source .venv/bin/activate

pip install -r requirements.txt
```

When you're ready for Stage 4, copy `.env.example` to `.env` and fill in
your Anthropic key:

```bash
cp .env.example .env
```

## The stages

Each stage is one file in `src/`, runnable on its own, building toward the
final agent.

| Stage | File | Concept | Needs API key? |
|---|---|---|---|
| 1 | `src/ingest.py` | Load docs → split → embed → store in Chroma | No |
| 2 | `src/rag_chain.py` | LangChain/LCEL retrieve-then-answer chain | Retrieval: no. Generation: yes |
| 3 | `src/tools.py` | `@tool`-decorated functions (calculator, datetime, web search) | No |
| 4 | `src/graph.py` | LangGraph agent: routes to RAG or tools, then generates | Yes |
| — | `src/chat.py` | Interactive CLI over the finished graph | Yes |

Run them in order:

```bash
python -m src.ingest        # builds chroma_db/ from data/docs/
python -m src.rag_chain     # retrieval demo (+ generation if you have a key)
python -m src.tools         # tool smoke tests
python -m src.graph         # full agent, two example questions
python -m src.chat          # interactive chat
```

Read the corresponding note in `data/docs/` alongside each stage:
`01_llms_and_prompts.md`, `02_langchain_basics.md`, `03_rag_pipeline.md`,
`04_tool_calling.md`, `05_langgraph_basics.md`, `06_agents_vs_chains.md`.

## Tests

```bash
python -m pytest tests/smoke_test.py -v
```

Covers ingestion, retrieval, and the tools — none of it needs an API key.

## Project layout

```
data/docs/       study notes = the RAG knowledge base
src/
  config.py      paths, model names, .env loading
  ingest.py      Stage 1
  rag_chain.py   Stage 2
  tools.py       Stage 3
  graph.py       Stage 4
  chat.py        CLI entry point
tests/
  smoke_test.py
chroma_db/       generated by ingest.py, gitignored
```

## Stretch ideas (not yet built)

- **Self-correcting retrieval**: grade retrieved docs for relevance and
  loop back to rewrite the query if they don't answer the question
  (a "Corrective RAG" pattern).
- **Conversation memory**: use a LangGraph checkpointer so the agent
  remembers earlier turns instead of treating each question independently.
- **Streaming**: stream tokens to the CLI instead of waiting for the full
  answer.
